# Augmentis Systems — Landing Page Design Library

## Purpose

This is the visual system and UI library for the Augmentis Systems company landing page.

It is designed to express:

- trust
- precision
- calm competence
- operational clarity
- premium restraint
- modern B2B seriousness

The page should feel like a company that understands systems, respects employees, and builds useful AI tools for German B2B / Mittelstand.

It should not feel loud, salesy, trend-chasing, or experimentally artistic.

---

# 1. Brand design premise

## Core visual thesis

Design Augmentis as a composed systems company.

Not a futuristic AI spectacle.
Not a consultancy brochure.
Not a startup cliché.

The visual language should sit at the intersection of:

- premium enterprise software
- editorial restraint
- product clarity
- quiet technical sophistication

## Emotional effect

A visitor should feel:

- this company is serious
- this company is thoughtful
- this company is modern
- this company is structured
- this company is safe to trust with important workflows

---

# 2. Foundation design principles

## 2.1 First principle

**Trust before intensity.**

Every visual choice should first help the page feel trustworthy.

## 2.2 Secondary principles

- Reduce noise.
- Increase legibility.
- Make structure visible.
- Use whitespace as a control signal.
- Prefer precision over decoration.
- Make complexity feel contained.
- Use contrast strategically, not aggressively.

## 2.3 Aesthetic standard

The page should feel:

- disciplined
- elegant
- calm
- minimal without being sterile
- premium without being luxurious in a flashy way

- overly dominant gradients
- decorative effects without semantic purpose

### Background color should be used for

- page base

### Neutral surfaces should be used for

- cards
- product frames
- section segmentation
- stat tiles
- principle blocks

---

# 4. Contrast philosophy

The design should not rely on hard black/white contrast everywhere.
It should use **soft but confident contrast**.

This means:

- warm base
- dark neutral text
- blue used as a controlled anchor
- borders visible but not harsh
- subtle separation between layers

The site should feel clear, not sharp-edged.

---

# 5. Typography system

## 5.1 Typography direction

Typography should carry a large part of the premium feeling.
It should feel editorial, technical, and highly legible.

The ideal impression is:

- modern
- composed
- intelligent
- exact
- understated

## 5.2 Recommended font pairing

### Primary sans
Use a neutral, premium grotesk or humanist sans.
Recommended options:

- **Inter**
- **Geist**
- **Söhne** if available in a brand environment
- **IBM Plex Sans** if a more technical tone is desired

### Optional display accent
Only if needed very sparingly:

- a refined serif for rare emphasis in one or two lines

But default recommendation:
**use one excellent sans only.**
This fits the brand better.

## 5.3 Type scale

### Display XL
- 64 / 72
- weight 600
- tracking -0.03em
- use: hero headline on desktop

### Display L
- 52 / 60
- weight 600
- tracking -0.03em
- use: primary headlines

### H1
- 40 / 48
- weight 600
- tracking -0.025em

### H2
- 32 / 40
- weight 600
- tracking -0.02em

### H3
- 24 / 32
- weight 600
- tracking -0.015em

### H4
- 20 / 28
- weight 600
- tracking -0.01em

### Body L
- 18 / 30
- weight 400

### Body M
- 16 / 28
- weight 400

### Body S
- 14 / 22
- weight 400

### Label L
- 14 / 20
- weight 500
- tracking 0

### Label S
- 12 / 16
- weight 600
- tracking 0.02em
- uppercase only when highly intentional

## 5.4 Typography rules

- Avoid overly bold blocks.
- Use weight contrast sparingly.
- Use spacing and hierarchy more than excessive font variety.
- Headlines should be concise and strong.
- Body text should remain easy to scan.
- Long paragraphs should not exceed comfortable reading width.

# 6. Layout system

## 6.1 Grid

### Desktop
- 12-column grid
- max width: `1200px`
- outer padding: `32px`
- column gap: `24px`

### Large desktop
- max width may extend to `1280px` when needed for product visuals

### Tablet
- 8-column logic
- outer padding: `24px`

### Mobile
- 4-column logic
- outer padding: `20px`

## 6.2 Section spacing

### Desktop rhythm
- section padding top/bottom: `104px` to `136px`
- tighter sub-sections: `64px` to `80px`

### Mobile rhythm
- section padding top/bottom: `72px` to `88px`

## 6.3 Content width guidance

- narrow reading column: `640px`
- standard content block: `720px`
- wide feature row: `960px`
- full content max: `1200px`

## 6.4 Layout principle

Use asymmetry carefully.
The site should not be mechanically centered everywhere.
But asymmetry must feel intentional and balanced.

---

# 7. Spatial system

## 7.1 Spacing scale

Use an 8pt system.

- 4
- 8
- 12
- 16
- 24
- 32
- 40
- 48
- 64
- 80
- 104
- 136

## 7.2 Radius system

- Small radius: `10px`
- Medium radius: `16px`
- Large radius: `24px`
- XL radius: `32px`

Recommended default for major cards:
**24px**

## 7.3 Border system

- standard border: `1px solid #D2CDC2`
- strong border: `1px solid #B8B2A7`
- emphasis border: `1px solid #B7C7FF`

Borders should remain subtle.
Use them to articulate layers, not to box everything harshly.

---

# 8. Shadow and depth system

Depth should feel quiet and architectural.
Not glossy.
Not floating excessively.

## 8.1 Shadow tokens

### Shadow 1
`0 1px 2px rgba(20, 23, 31, 0.04)`

### Shadow 2
`0 6px 20px rgba(20, 23, 31, 0.06)`

### Shadow 3
`0 16px 40px rgba(20, 23, 31, 0.08)`

### Shadow 4
`0 24px 60px rgba(20, 23, 31, 0.10)`

## 8.2 Depth rules

- use shadow more on hero visual and elevated cards
- use borders more than shadows for most UI
- do not combine aggressive shadows with strong borders unless the component is a focal point

---

# 9. Component philosophy

Components should feel like software-grade objects.
Not decorative website elements.

They should communicate:

- structure
- clarity
- modularity
- control

Each component should look like it belongs to a system.

---

# 10. Component library

## 10.1 Navigation bar

### Structure
- left: Augmentis wordmark
- center/right: minimal nav links
- far right: primary CTA

### Style
- transparent or warm elevated background
- thin bottom separation on scroll
- compact vertical height
- minimal visual clutter

### Nav links
- Body S or Label L
- dark neutral default
- blue hover

### CTA button in nav
- primary filled button

---

## 10.2 Buttons

### Primary button
Use for main CTA.

- background: `#002FA7`
- text: `#F8F7F4`
- radius: `999px` or `16px`
- padding: `14px 20px`
- hover: `#1A45B8`
- shadow: none or very subtle

Tone: decisive, polished, not oversized.

### Secondary button
Use for alternative CTA.

- background: transparent or `#F6F4EF`
- text: `#14171F`
- border: `1px solid #D2CDC2`
- hover surface: `#E7E3DB`

### Tertiary / text button
Use for inline exploration.

- no heavy chrome
- text dark or blue
- underline on hover optional

### Button rules

- avoid giant SaaS-pill buttons that feel overly consumer
- prefer medium width and high clarity
- labels should be direct and calm

---

## 10.3 Section labels / eyebrow labels

Use to frame sections with precision.

- size: Label S
- color: brand deep or text secondary
- background: brand surface or neutral surface
- radius: `999px`
- padding: `6px 10px`

Use sparingly.
They should feel like orientation markers, not badges for decoration.

---

## 10.4 Cards

### Principle cards
For values, principles, or claims.

- background: `#F6F4EF`
- border: `1px solid #D2CDC2`
- radius: `24px`
- padding: `24px` to `32px`

### Feature cards
For process or capability blocks.

- same base as principle cards
- optional small blue accent element
- title + body + optional mini diagram

### Product cards
For KlarOrder showcase or future products.

- background: `#F8F7F4` or `#EEF3FF`
- border: soft neutral or brand border
- slightly stronger depth allowed

### Card rules

- avoid stuffing cards with too much text
- one clear concept per card
- use cards to create modular logic, not to fragment everything

---

## 10.5 Metrics / stat tiles

For showing workflow improvements, principles, or concise proof points.

- large number or phrase
- restrained label beneath
- high whitespace
- left-aligned
- no over-gamified styling

Should feel like quiet evidence.

---

## 10.6 Product preview frame

This is important.
The product visual should not look like a random Dribbble mockup.
It should look like a believable enterprise workflow interface.

### Product frame style
- warm-white or cool-neutral surface
- subtle border
- large radius `24px` to `32px`
- layered internal panels
- thin dividers
- realistic density
- restrained blue highlights for key actions and selected elements

### Product preview content style
- use tables, review panels, sidebars, tags, status rows
- favor operational clarity over glossy illustration
- use realistic information hierarchy

This helps communicate that Augmentis builds software for real processes.

---

## 10.7 Iconography

### Icon style
- simple
- geometric
- line-based or duotone minimal
- not playful
- not skeuomorphic

### Icon usage
- support comprehension
- break up dense copy
- show system qualities like review, structure, flow, control

### Avoid
- overly colorful icons
- emoji-like iconography
- novelty illustration icons

---

## 10.8 Dividers and separators

Use separators lightly.

- 1px soft border lines
- generous space above and below
- can be paired with background changes instead of always using visible rules

---

## 10.9 Forms

If a contact form is used:

- high-legibility fields
- soft borders
- generous padding
- clear labels
- no excessive placeholder dependence
- serious and calm tone

### Input style
- background: `#F8F7F4`
- border: `1px solid #D2CDC2`
- radius: `16px`
- height: `48px` to `56px`
- focus: blue outline or blue border with soft glow

---

# 11. Motion system

Motion should feel deliberate and understated.

## 11.1 Motion characteristics

- smooth
- quiet
- short to medium duration
- no bouncy consumer app motion
- no exaggerated parallax

## 11.2 Recommended timings

- hover: `120–180ms`
- reveal: `320–500ms`
- modal/drawer if any: `260–360ms`

## 11.3 Motion rules

Use motion to:

- guide attention
- confirm interaction
- support hierarchy
- make the interface feel refined

Do not use motion to:

- impress for its own sake
- create theatrical AI atmosphere
- distract from content

---

# 12. Imagery and illustration direction

## 12.1 Preferred imagery modes

The strongest visual modes for this brand are:

- abstract systems diagrams
- interface previews
- structured operational visuals
- subtle editorial compositions

## 12.2 If using photography

Photography should be used rarely.
If used, it should show:

- real work environments
- calm professional settings
- no forced stock-photo smiling team clichés

## 12.3 Illustration style

If illustration exists, it should be:

- diagrammatic
- restrained
- architectural
- abstract enough to feel premium

Avoid whimsical illustration styles.

# 13. Section-specific design patterns

## 13.1 Hero section

### Goal
Immediate trust, clarity, and brand distinction.

### Structure
- left: headline, supporting copy, CTA row
- right: product/system visual or structured abstract composition

### Design cues
- strong typographic hierarchy
- large negative space
- one focal blue accent system
- no visual clutter

### Hero visual options
1. A refined pseudo-product interface showing AI-assisted workflow review
2. A systems diagram with structured flow blocks
3. A hybrid of interface and operational flow

Preferred option:
**hybrid of interface and system logic**

---

## 13.2 Philosophy / principles section

### Pattern
Use 3–4 principle cards in a grid.

### Content style
- one short title
- short explanation
- optional icon or micro visual

Should feel like the company’s operating convictions.

---

## 13.3 Product proof section

### Pattern
Large featured module for KlarOrder.

Potential structure:
- left: product explanation
- right: product preview
- beneath: 3 supporting benefits or workflow qualities

The product module should feel more concrete and slightly more detailed than the rest of the page.

---

## 13.4 Why this matters for Mittelstand section

### Pattern
Editorial text paired with stat or insight blocks.

This section can be visually quieter.
It should feel reflective and credible.

---

## 13.5 Trust / implementation section

### Pattern
Process row or 3-step adoption blocks.

Visual tone:
- calm
- ordered
- procedural

This section reduces perceived risk.

---

## 13.6 CTA section

### Pattern
Contained high-clarity closing block.

- elevated surface
- strong but concise headline
- one primary CTA
- optional secondary CTA

Should feel like a safe, logical next step.

# 14. Wordmark and logo behavior

If the logo is text-based or minimal:

- default wordmark in text primary or brand deep
- avoid over-designed symbol systems unless highly refined
- spacing around logo should feel generous

The wordmark should feel like a software company, not a luxury brand.

---

# 15. Accessibility rules

This system should remain accessible and usable.

## Requirements
- body text must maintain strong enough contrast
- blue text on warm background should be used carefully for long text blocks
- interactive states must be visible beyond color only
- buttons must be sufficiently sized
- focus states must be explicit
- mobile spacing must remain generous

## Specific caution
`#002FA7` on `#F0EEE9` is generally strong enough for many headline and button use cases, but long passages should usually remain in dark neutral text for reading comfort.

---

# 16. Responsive behavior principles

On mobile:

- preserve hierarchy, do not flatten everything
- keep hero strong but concise
- stack modules with generous spacing
- simplify product preview to one strong image/frame
- preserve button clarity and comfortable tap targets

The mobile version should feel like a distilled, premium version of the desktop experience, not a compressed afterthought.

---

# 17. Design anti-patterns

Do not use:

- glowing neon effects
- AI particle visuals
- excessive gradients
- glassmorphism as a dominant system
- hard pure white everywhere
- harsh black blocks unless extremely intentional
- gimmicky animated lines
- oversized marketing badges
- crowded comparison grids in the first impression
- playful startup illustrations
- generic corporate stock people photography as a core asset

---

# 18. Design system summary sentence

The Augmentis Systems landing page should look like a premium, calm, highly structured B2B software brand built for serious German businesses — warm in tone, precise in hierarchy, restrained in color, and quietly confident in execution.
